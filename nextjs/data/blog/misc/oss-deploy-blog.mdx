## Introduction

This blog post is about the motivation behind deploying a blog using **OSS services**. The blog is built using Next.js and hosted on GitHub.io. The blog content is stored in a GitHub repository. The blog is deployed using GitHub Actions. Here everything is ok.

But we cant easy to touch the Github.io in the _china mainland_. So we need to deploy the blog to the OSS service.
Let more people can access the blog.

### What is OSS Service ?

The OSS **Object Storage Service** is a distributed storage service provided by Alibaba Cloud or etc. It is designed to store large amounts of data at low costs. The OSS service is highly scalable and can be used to store a wide range of data types, including images, videos, and other multimedia files. Here we use the OSS service to store the blog content and assets.

## Deploy Blog to OSS

On the past, we talk about how to deploy the [static blog](statisc-blog) to the Github.io. On those post, we build our blog using Next.js and store the blog content in a GitHub repository. We use GitHub Actions to deploy the blog to GitHub.io. The blog static files is builded by the Next.js and store in the `out` directory.

For integrating the OSS service, we need to upload the `out` directory to the OSS service. We can use the `ossutil` tool to upload the files to the OSS service. The `ossutil` is a command-line tool provided by Alibaba Cloud that allows you to manage your OSS buckets and objects. we are easy to use the `ossutil` to upload the files to the OSS service on the GitHub Actions.

```yaml:deploy.yml
deploy-oss:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: github-pages
          path: artifact
      - name: Extract artifact
        run: |
          mkdir -p dist
          cd artifact
          tar -xf artifact.tar -C ../dist
      - uses: manyuanrong/setup-ossutil@v3.0
        with:
          endpoint: "oss-cn-guangzhou.aliyuncs.com"
          access-key-id: ${{ secrets.OSS_ACCESS_KEY_ID }}
          access-key-secret: ${{ secrets.OSS_ACCESS_KEY_SECRET }}
      - run: ossutil cp -rf dist/ oss://isomo/ --meta=Cache-Control:no-cache
```
